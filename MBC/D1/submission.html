Evolution of data transfer protocols:

Alike any other field of technology there has been a significant progress made in the data transfer protocols in terms of speed, scrutiny and security.

HTTP1.1 HTTP2.0
Sends data in a single request and waits for confirmation before sending the next set of requests. Each request requires a new connection. Allows multiple data requests to be sent over a single TCP connection, reducing latency.
Server does not proactively respond to requests not made by the client; waits for client's request. Supports server push, allowing servers to proactively push resources to clients before they are requested.
Does not compress header information; data is presented as plain text. Compresses header data using HPACK for quick data movement.
Does not prioritize important data. Prioritizes important resources of data.
Transmits data in textual form. Transmits data in binary form, allowing smoother and efficient data transmission.
Data is not encrypted. Supports TLS encryption, ensuring secure data transmission.
